# ShuFu v0.1 设计

## 1. 目标

v0.1 验证最小闭环：Windows/Linux 安装 ShuFu 后可以直接使用模型、启动本地模型节点、保存连续会话，并把会话和产物交给后续 Android 客户端继续使用。

首批平台边界固定为 Windows/Linux、Android、ESP32。v0.1 实现桌面端；Android 和 ESP32 共享的协议、记忆包格式与工具边界在本版本冻结草案。

## 2. 非目标

- 不做账号系统、RBAC、证书中心和控制台；
- 不做云端托管、计费、模型市场；
- 不做向量数据库、RAG 和长期人格；
- 不允许 Agent 执行任意 Shell 或下载未知代码；
- 不试图在 ESP32 本地运行大语言模型。

## 3. 极简架构

```text
CLI / Android / ESP32
          |
   ShuFu v0.1 Protocol
          |
      ShuFu Node
      /        \
  Runtime     MemoryStore
  /  |  \       /     \
echo llama API sessions artifacts
```

核心边界：

1. `Runtime` 只负责根据消息生成文本；
2. `MemoryStore` 只负责会话、消息和文档产物；
3. `Node` 组合运行时和记忆，不依赖具体模型；
4. `Client` 只理解稳定 HTTP/JSON 协议；
5. `ToolRegistry` 是 Agent Lite 的安全扩展点，不进入默认推理流程。

## 4. 记忆连续性

记忆分为三层：

### 4.1 会话记忆

SQLite 保存 `session -> messages`。每次调用读取最近 N 条消息，因此同一 session 可以连续交谈。

### 4.2 产物记忆

生成的 `.md`、文本或其他文档以 SHA-256 内容寻址保存。数据库记录：

- 产物 ID；
- 会话 ID；
- 文件名和 MIME；
- 内容哈希及大小；
- 创建时间。

`shufu run --save-output notes.md` 会同时写出文件和加入产物索引。

### 4.3 跨设备同步

v0.1 使用可移植 JSON Bundle 和 HTTP 导入/导出接口。Bundle 包含会话、消息、产物元数据和 Base64 内容。未来 Android 只需实现同一 Bundle，即可接续 Windows 产生的会话和 Markdown。

前期刻意不做自动云同步。原因是冲突解决、账户、端到端加密和后台同步会显著放大复杂度。后续可在 Bundle 之上增加：

- 增量游标；
- 点对点局域网同步；
- 用户自选 WebDAV/Git/对象存储；
- 语义摘要与可插拔向量索引。

数据库和协议均保留 `schema_version`，避免未来升级破坏旧记忆。

## 5. 记忆风险与约束

| 风险 | v0.1 处理 |
| --- | --- |
| 文档泄露 | 默认仅本机存储，导出必须显式执行 |
| 记忆无限增长 | 推理仅加载最近窗口，产物不自动进入提示词 |
| 错误记忆污染 | 原始消息与未来摘要分离，v0.1 不自动写长期事实 |
| 多端冲突 | 记录不可变 ID 和时间；v0.1 合并去重，不覆盖原记录 |
| 大文件传输 | Bundle 用于轻量文档；后续增加分块和外部对象引用 |
| 提示注入藏在文档中 | 产物只是数据，不自动当系统指令执行 |

## 6. 极简权限

v0.1 不实现完整权限验证模块。

只保留三条规则：

1. Node 默认绑定 `127.0.0.1`，本机调用不鉴权；
2. 非回环地址必须显式传入 `--allow-lan`；
3. 局域网可选一个静态 Bearer Token。

没有用户、角色、证书、设备注册、策略服务或外部身份依赖。这样不会影响核心运行稳定性，同时避免默认把模型和记忆暴露到局域网。

后续只有在真实社区需求出现时，再以独立适配器增加 mTLS、OAuth 或设备配对。

## 7. Agent Lite 预留

v0.1 提供 `ToolRegistry`、工具元数据、参数对象校验、超时和最大步数配置，但默认不运行自主循环。

后续 Agent 记忆直接复用：

- 会话消息作为短期上下文；
- Artifact 作为用户显式选择的工作资料；
- 未来 Summary/Fact Store 作为可选长期记忆；
- 同一 session ID 在 Windows、Android、ESP32 之间传递。

工具执行坚持：只运行宿主预注册函数，不运行模型生成的任意代码。

## 8. v0.1 验收标准

- `shufu run` 零外部依赖可运行；
- 可选 GGUF 和 OpenAI-compatible runtime 可配置；
- `shufu serve` 默认仅本机监听；
- HTTP 调用保存并继续同一会话；
- Markdown 产物可以导出、导入并保持哈希一致；
- 单元测试覆盖记忆、协议、节点、权限边界和工具注册；
- Windows/Linux 使用 Python 3.10+ 运行。
